# React Native Screen Builder Skill

Use this workflow for React Native screen work.

## Step 1: Understand Existing Patterns

Before editing:

- Find existing screens.
- Find navigation setup.
- Find shared components.
- Find API client pattern.
- Find styling conventions.

## Step 2: Define Screen Requirements

Clarify:

- Screen purpose
- Input data
- API endpoints needed
- Loading state
- Empty state
- Error state
- Success behavior
- Navigation behavior

## Step 3: Build Safely

When implementing:

- Use TypeScript.
- Keep component readable.
- Use FlatList for lists.
- Keep API calls in hooks/services when possible.
- Avoid adding libraries unless necessary.
- Match existing folder structure.

## Step 4: Verify

After implementation:

- Check TypeScript errors.
- Check navigation route names.
- Check API request/response types.
- Check loading/error/empty states.
- Explain changed files.
