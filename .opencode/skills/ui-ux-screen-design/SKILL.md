# UI/UX Screen Design Skill

Use this skill for all professional mobile UI/UX work.

## Step 1: Inspect Existing Design

Before changing UI, inspect:

- Existing screens
- Shared components
- Color system
- Typography system
- Spacing patterns
- Button styles
- Input styles
- Card/list styles
- Navigation/header patterns

Do not create a new visual style without checking existing patterns.

## Step 2: Define Screen Purpose

For each screen, identify:

- What is the user trying to do?
- What is the primary action?
- What is secondary?
- What information is most important?
- What can be hidden, delayed, or simplified?

## Step 3: Apply UI Hierarchy

Improve:

- screen title
- sections
- primary action
- secondary action
- content grouping
- empty state
- error state
- loading state
- success feedback

## Step 4: Apply Consistency

Check:

- spacing
- padding
- typography
- colors
- border radius
- shadows
- buttons
- inputs
- list items
- icons
- cards
- modals

Use shared components and design tokens when available.

## Step 5: Mobile UX Quality

Ensure:

- comfortable touch targets
- safe area support
- keyboard handling
- scroll behavior
- responsive layout
- readable text
- clear feedback
- accessible labels where useful

## Step 6: Keep Code Simple

When implementing UI:

- Avoid overengineering.
- Avoid unnecessary libraries.
- Avoid random inline styles.
- Prefer shared styles/components.
- Keep screens readable.
- Extract reusable components only when repeated.

## Step 7: Final Report

Always report:

1. UI/UX changes made
2. Screens/components changed
3. Consistency improvements
4. States handled
5. How to visually test
6. Remaining design risks
