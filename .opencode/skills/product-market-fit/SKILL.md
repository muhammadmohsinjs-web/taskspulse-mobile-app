# Product Market Fit Skill

Use this skill when the user asks product, market, business, startup, monetization, positioning, roadmap, or idea validation questions.

## Step 1: Understand the Product

Inspect available project files if needed:

- README.md
- docs/
- package.json
- frontend screens
- backend APIs
- feature names
- existing business idea documents

Identify:

- What the product does
- Who it is for
- What problem it solves
- What features already exist
- What features are planned

## Step 2: Define the User

Answer:

- Who has the strongest pain?
- Is this for personal use, teams, students, developers, founders, agencies, or businesses?
- Who would pay?
- Who would use it daily or weekly?

## Step 3: Pain Analysis

Rate the pain:

- Urgent or casual?
- Frequent or rare?
- Expensive or annoying?
- Already solved or underserved?
- Must-have or nice-to-have?

## Step 4: Competitor and Alternative Analysis

Compare against:

- Direct competitors
- Indirect competitors
- Manual workflows
- Spreadsheets
- Existing apps
- AI tools
- Open-source tools

If web access is available, research competitors.
If web access is not available, clearly mark this as assumption-based.

## Step 5: MVP Scope

Recommend:

- Core MVP features
- Features to delay
- Features to remove
- First user journey
- First 7-day build plan
- First validation test

## Step 6: Monetization

Evaluate:

- Free personal use
- Freemium
- Paid subscription
- Lifetime deal
- Team plan
- B2B plan
- Usage-based pricing
- Marketplace/add-on model

## Step 7: Product-Market Fit Score

Give scores from 1 to 10:

- Pain strength
- User clarity
- Market demand
- Differentiation
- Monetization potential
- Distribution difficulty
- Retention potential
- Build complexity

## Step 8: Final Recommendation

End with one of:

- Build MVP
- Niche down
- Pivot positioning
- Validate before building
- Stop or park the idea

Always explain why.
